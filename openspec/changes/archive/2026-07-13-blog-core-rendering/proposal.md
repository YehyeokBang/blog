## Why

현재 블로그 프로젝트의 인프라(Next.js, 배포 파이프라인) 구축이 완료되었으나, 실제 블로그 콘텐츠인 "마크다운 글"을 화면에 보여주는 핵심 렌더링 로직이 부재한 상태입니다. 사용자가 마크다운 파일과 이미지를 작성하면 이를 실제 웹페이지에서 읽고 볼 수 있도록 MVP의 핵심 가치인 "블로그 본연의 기능"을 구현해야 합니다.

## What Changes

- **마크다운 파싱 엔진 구현**: `gray-matter`, `zod`, `remark`, `rehype` 생태계를 활용하여 `content/posts/*.md` 파일을 HTML로 변환하고, 강력한 Frontmatter 스키마 검증을 거칩니다.
- **코드 하이라이팅 적용**: `shiki` 및 `rehype-pretty-code`를 통해 개발자 블로그에 필수적인 코드 하이라이팅 테마를 입힙니다.
- **정적 이미지 서빙 방식 확립**: Next.js의 `public/images/` 디렉토리를 활용하여 마크다운 내부에서 정적 이미지를 자연스럽게 불러올 수 있는 구조를 마련합니다.
- **메인 및 상세 페이지 UI 구현**: 글 목록을 보여주는 피드 페이지와 개별 글을 읽을 수 있는 상세 페이지 컴포넌트를 개발합니다.

## Capabilities

### New Capabilities
- None

### Modified Capabilities
- `blog-rendering`: 실제 마크다운 파싱 로직, 코드 하이라이팅, 정적 이미지 로드 및 메인/상세 페이지 뷰어에 대한 세부 시나리오를 추가합니다.

## Impact

- `content/posts/` 경로에 작성된 마크다운 데이터가 Next.js 앱의 라우팅(`/posts/[slug]`) 및 빌드 프로세스에 직접적인 영향을 미칩니다.
- 이미지 파일 관리는 Next.js의 정적 파일 제공 폴더인 `public/`에 집중됩니다.
