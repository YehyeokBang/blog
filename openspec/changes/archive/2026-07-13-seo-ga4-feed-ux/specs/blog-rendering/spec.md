## MODIFIED Requirements

### Requirement: 마크다운 정적 파싱 및 피드 렌더링
시스템(Next.js)은 빌드 타임에 `content/posts/` 디렉토리 내의 마크다운 파일 목록을 읽어 최신순으로 정렬된 정적 HTML 블로그 피드 페이지를 생성해야 한다.

#### Scenario: 피드 목록 정적 생성
- **WHEN** 빌드 스크립트(`next build`)가 실행될 때
- **THEN** 시스템은 각 마크다운의 Frontmatter(date, title, tags 등)를 파싱하여 메인 피드 페이지의 HTML 정적 파일을 생성한다.

#### Scenario: 피드 목록 내 이미지 썸네일 노출
- **WHEN** 마크다운 파일의 Frontmatter에 `thumbnail: /images/...` 속성이 존재할 때
- **THEN** 시스템은 메인 피드 목록에서 해당 게시글의 카드 우측에 썸네일 이미지를 렌더링한다. 이미지는 고정 크기로 `object-fit: cover` 처리되어 비율을 유지하며 잘라낸다. 단, 모바일 해상도(sm 미만)에서는 썸네일을 숨겨 텍스트 가독성을 보장해야 한다.

#### Scenario: 피드 카드 전체 클릭 영역
- **WHEN** 사용자가 피드 목록의 카드 영역(제목, 설명, 날짜 등 카드 내 임의의 위치) 중 태그 버튼을 제외한 영역을 클릭할 때
- **THEN** 시스템은 해당 포스트의 상세 페이지(`/posts/[slug]`)로 이동해야 한다.

#### Scenario: 태그 버튼 독립 동작
- **WHEN** 사용자가 피드 카드 내의 태그 버튼을 클릭할 때
- **THEN** 시스템은 해당 태그로 필터링 동작을 수행하며, 포스트 상세 페이지로의 이동은 발생하지 않아야 한다.
