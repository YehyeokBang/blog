# 페이지 헤더와 소개 섹션 리듬 디자인

## 배경

아티클, 소개, 경력, 활동 제목은 같은 화면 폭 안에서 서로 다른 간격 규칙을 사용한다. 페이지 제목은 명시적인 하단 간격과 line-height가 없고, 소개 페이지의 주요 구획 간격은 `소개 → 경력`이 80px인 반면 `경력 → 활동`은 32px이다. 또한 페이지 제목과 하위 섹션 제목이 같은 크기와 굵기를 사용해 정보 위계가 약하다.

## 목표

- 아티클과 소개 페이지 제목에서 첫 콘텐츠까지의 간격을 24px로 통일한다.
- 모든 제목에 명시적인 `leading-tight`를 적용해 글꼴 line box에 따른 시각적 편차를 줄인다.
- 소개 페이지의 경력과 활동을 페이지 제목보다 한 단계 낮은 시각 위계로 표현한다.
- 소개, 경력, 활동의 주요 구획 사이는 80px로 통일한다.
- 현재 블로그의 여백 중심·최소 장식 디자인을 유지한다.

## 디자인 결정

### 페이지 제목

아티클과 소개의 `h1`은 기존 크기인 모바일 28px, 데스크톱 40px과 `font-extrabold`를 유지한다. `leading-tight`와 `mb-lg`를 적용해 제목 line-height와 다음 콘텐츠까지의 24px 간격을 명시한다.

### 소개 페이지 섹션 제목

경력과 활동의 `h2`는 모바일 22px, 데스크톱 28px로 조정한다. `font-extrabold`, `leading-tight`, `tracking-tight`, `mb-lg`를 공통 적용한다. 페이지 제목보다 작지만 각 섹션의 시작점으로 충분한 강조를 유지한다.

### 주요 구획 간격

소개와 경력, 경력과 활동 사이는 모두 `spacing-section`인 80px를 사용한다. `HistorySection` 자체의 임의 하단 margin을 제거하고 두 이력 섹션을 `gap-section` 컨테이너로 묶어 간격의 단일 책임을 부모에 둔다.

### 유지하는 요소

- 아티클 태그 pill, 하단 hairline과 목록 시작 간격
- 소개 본문, 연락처와 이력 항목의 타이포그래피
- 색상, surface, radius와 전체 1000px shell
- 모바일·데스크톱 breakpoint

카드, 배경 surface, 추가 구분선은 도입하지 않는다. 현재 디자인 시스템이 장식보다 여백과 타이포그래피를 우선하기 때문이다.

## 검증

이 변경은 CSS class 조정만 포함하므로 별도의 로직 회귀 테스트를 추가하지 않는다. ESLint, 기존 프론트엔드 테스트 18개, Next.js 정적 빌드로 문법·타입·렌더 가능 여부를 검증한다. 사용자의 요청에 따라 별도 브라우저 QA는 수행하지 않고, 완료 후 개발 서버를 실행해 사용자가 직접 확인한다.
