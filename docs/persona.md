## 글쓰기 스타일

인터뷰와 Velog 42개 글 분석을 통해 도출한 글쓰기 패턴:

### 스토리텔링형 기술 글
```
[대화체 도입] "우리 팀은 ~한 문제를 만났어요"
     ↓
[문제 정의] 구체적 수치와 함께 상황 설명
     ↓
[탐색 과정] 여러 선택지 비교 + 왜 이걸 선택했는지
     ↓
[구현 & 검증] 코드, 쿼리 실행 계획, 성능 수치 제시
     ↓
[회고 & 교훈] 배운 점, 아쉬운 점 솔직하게 서술
```

### 특징
- **대화체 + 전문성**: "~했어요" 말투에 실제 코드와 수치 데이터가 탄탄하게 뒷받침
- **팀 경험 기반**: 개인 TIL이 아닌, 실제 프로젝트에서 부딪힌 문제를 다룸
- **깊이 있는 분석**: 단순 "이렇게 해봤다"가 아니라 "왜 이걸 선택했는가"까지 서술
- **시각 자료 적극 활용**: 아키텍처 다이어그램, 쿼리 실행 계획 표, 코드 블록 다수
- **긴 글**: 코드 포함 2000~5000자. 가벼운 TIL보다는 심층 분석형

### 블로그 설계에의 시사점
- **긴 글** → TOC 사이드바, 스크롤 프로그레스 바 필수
- **코드 블록** 핵심 → Syntax Highlighting + 복사 버튼 필수
- **시리즈 개념** 활용 → 시리즈 내 이전/다음 네비게이션
- **다이어그램/이미지** 적극 사용 → 이미지 최적화, 레이지 로딩
- **대화체 + 전문성** → 깔끔하되 친근한 디자인 톤
